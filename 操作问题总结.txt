Gangge Code 操作问题总结
========================

编写日期: 2026-05-14
对比对象: Claude Code / Cursor 等同类 AI 编程助手


一、当前问题
──────────

1.1 AI 不执行，只说话
────────────────────
  用户输入任务（如"帮我写一个SaaS系统"）后，
  AI 回复文字："我来先了解一下当前工作目录的结构，然后开始构建..."
  然后什么都不做，没有调用任何工具（list_dir / write_file / bash等）。

1.2 用户看不到进度
────────────────
  AI 干活的过程完全没有可视化反馈：
  - 不知道 AI 在"思考"还是在"等 API"
  - 没有步骤提示（第几步、做什么）
  - 没有任务清单 / todo list
  - 用户看到的只有一句蓝色文字，然后"卡住"

1.3 无法区分"卡住"和"正在工作"
─────────────────────────────
  - 系统在等 LLM 回复时，UI 没有任何进度指示
  - 用户以为程序死了，实际上可能在等 API
  - 重试机制内部执行，用户看不到

1.4 系统提示（System Prompt）逻辑有误
───────────────────────────────────
  当前的 prompt 要求 AI：
    "先用 list_dir 查看目录结构"
    "先理解再行动"
  但对于从零开始的新项目（空文件夹），"探索"没有意义。
  正确的逻辑应该是：先规划输出 → 再逐步执行。


二、对比别人的工作模式（以用户提供的记录为例）
────────────────────────────────────────

2.1 别人的工作流程
────────────────
  接收任务后，AI 自动执行以下步骤（可见）：

  ① 分析需求并输出规划
     "The workspace is empty. Let me plan and build..."
     "Key modules: 1.Dashboard 2.Customer Management 3...."
     "Let me create the database models: 1.User 2.Customer..."
     → 用户看到 AI 在认真思考，有具体输出

  ② 创建任务清单（Todo List）
     "任务清单:
      1/7 创建项目结构和数据库模型
      2/7 实现认证系统
      3/7 ..."
     → 用户看到完整的步骤规划，知道要做什么

  ③ 逐步执行，每步可见
     "Let me start building this now."
     "创建后端包初始化文件"（实际调用 create_file）
     "任务清单 1/7 已完成"
     → 用户看到文件被逐个创建，进度在推进

2.2 别人的关键特征
────────────────
  ✓ 首轮必定输出有价值的规划内容（不是"我来了解一下"）
  ✓ 有任务清单（todo list），用户知道整体计划
  ✓ 每步完成后更新进度（N/M 已完成）
  ✓ 用户随时知道"进行到哪一步了"
  ✓ 即使是大项目，也能拆解成小步骤逐步执行


三、根本原因分析
──────────────

3.1 操作逻辑不对
──────────────
  当前系统让 AI "先理解再行动"，但对于新项目，
  空文件夹没什么好理解的。应该改为：
    "先规划输出 → 再逐步执行"

3.2 系统提示太弱
──────────────
  - 没有要求 AI 首轮必须输出规划和模块清单
  - 没有要求 AI 创建 todo list
  - 没有要求 AI 每步更新进度
  - "不要只回复文字"不够强硬

3.3 用户反馈缺失
──────────────
  - LLM 调用期间无任何进度指示
  - 重试/纠偏过程不对用户展示
  - 工具执行结果展示太简略


四、改进方向（待讨论实施）
──────────────────────

4.1 System Prompt 重写
────────────────────
  改为三段式操作逻辑：

  【第一段：规划输出】
    分析需求 → 列出模块清单 → 设计数据模型
    （首轮必须输出这些，让用户看到计划）

  【第二段：任务清单】
    创建步骤化 todo list
    格式：
      任务清单 N/M:
      1. xxx — 涉及文件: ...
      2. xxx — 涉及文件: ...
      ...

  【第三段：逐步执行】
    每轮执行一个或多个步骤
    完成后更新进度: "✅ 2/7 已完成"
    全部完成后输出总结

4.2 进度可视化
────────────
  - LLM 调用期间显示 "⏳ 正在分析..."
  - 每一步执行结果显示在当前会话中
  - 失败时显示具体错误原因

4.3 强制执行的兜底逻辑
────────────────────
  - 首轮无工具调用 → 追加强制指令重试（已有，需强化）
  - 多次重试无效 → 提示用户更换模型
